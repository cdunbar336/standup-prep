Connect Slack — Reading messages

To read channel history, you need to create a small Slack app in your workspace. It takes about 3 minutes and you don't need to be a Slack admin.

Create your Slack app and get a bot token:

  1. Go to api.slack.com/apps and click Create New App → From scratch
  2. Name it "Standup Prep" and choose your workspace
  3. In the left sidebar, click OAuth & Permissions
  4. Under Bot Token Scopes, add these three:
    • channels:history — read messages
    • channels:read — list channels
    • users:read — show names
  5. Scroll up and click Install to Workspace, then Allow
  6. Copy the Bot User OAuth Token (starts with xoxb-)
Bot token should start with xoxb-
Add at least one channel to monitor

Type a channel name (without #) and press Enter or comma to add it. Then run /invite @Standup Prep in each channel so the bot can read it.

Connect Slack — Posting the brief

The morning brief gets posted to a Slack channel via a webhook. Choose which channel you want it delivered to (usually your standup channel).

Add a webhook to your Slack app:

  1. Go back to your app at api.slack.com/apps and select Standup Prep
  2. In the left sidebar, click Incoming Webhooks
  3. Toggle Activate Incoming Webhooks on
  4. Click Add New Webhook to Workspace
  5. Choose the channel where you want the brief to appear
  6. Click Allow, then copy the Webhook URL
Webhook URL should start with https://hooks.slack.com/
🤖

Connect Claude AI

Claude does the reasoning — reading the Jira tickets and Slack messages and turning them into an actionable brief. You'll need an API key from Anthropic.

Get your Anthropic API key:

  1. Go to console.anthropic.com and sign in (or create a free account)
  2. Click API Keys in the left sidebar
  3. Click Create Key, name it "Standup Prep", and copy it
💡 The API costs a few cents per run. For a daily brief, typical usage is well under $1/month.
API key should start with sk-ant-
